TOGGLE COMPONENT
================

OVERVIEW
--------
A binary on/off control that gives users immediate feedback of a state
change without requiring a form submission. Used to toggle features,
preferences, and settings on/off.

Supports 8 visual states across 2 values (On / Off) and 4 interaction
states (Default, Hovered, Pressed, Disabled).


DIMENSIONS
----------
Track       35 × 20 px, border-radius: 200px (full pill)
Thumb       16 × 16 px, border-radius: 50%, offset 2px from near edge
Thumb — Off position   left: 2px
Thumb — On  position   left: 17px
Transition  background 0.18s ease, left 0.18s ease

CSS CLASSES (static / display use)
------------------------------------
.ds-toggle                 Base pill track — 35×20px, Off Default state
.ds-toggle__thumb          White circle inside the track
.ds-toggle--on             On state — blue track (#0c8dea), thumb slides right
.ds-toggle--hovered        Hover appearance override (for static demos)
.ds-toggle--pressed        Pressed appearance override (for static demos)
.ds-toggle--disabled       Muted track, no pointer, thumb shadowless

Combine modifiers as needed:
  .ds-toggle--on.ds-toggle--hovered   → On + Hovered
  .ds-toggle--on.ds-toggle--pressed   → On + Pressed  (navy #093C6B)
  .ds-toggle--on.ds-toggle--disabled  → On + Disabled (light blue #70AECE)


CSS CLASSES (interactive / real checkbox)
------------------------------------------
.ds-toggle-input           Native <input type="checkbox"> styled to match.
                           Use for real interactivity — hover, :checked,
                           :active, :disabled states are CSS-driven.

.ds-toggle-label           Flex label wrapping .ds-toggle-input + text
.ds-toggle-label--disabled Muted label text (#9e9e9e) for disabled rows

USAGE GUIDELINES
----------------
- Use a Toggle for settings that take effect immediately (no Save button needed).
- Always pair with a concise label that describes what the On state enables.
- Do not use a Toggle in place of a Checkbox inside a form that requires
  explicit submission — use Checkbox instead.
- Disabled toggles should still show the current value (on/off) so the user
  understands the locked state.
- Keep label text short — ideally a noun phrase (e.g. "Gap Analysis",
  "Show Competitors", "Weekly Digest").
