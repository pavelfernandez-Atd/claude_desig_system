ACCORDION COMPONENT
Adthena Design System — Component Specification
Figma source: node 214:1413 (Claude-DS file)
================================================


OVERVIEW
--------
The Accordion is a collapsible row component used in settings panels and ordered
configuration lists. Each row combines five interactive elements: a drag handle for
reordering, a toggle switch for enabling/disabling the item, a title + description
block, an optional count badge, and a circular expand button that reveals a content
panel below the row.

Typical use cases:
  • Ad platform configuration lists (Google Ads, Microsoft Advertising, Meta Ads…)
  • Filter or dimension ordering panels
  • Feature toggle lists within a settings sidebar

  Border:         1px solid #d3d3d3

  Padding:        16px (all sides)

  Font family:    'Open Sans', Helvetica, sans-serif

⑤ Expand button  (.acc__btn)
  Overall size:  32 × 32 px

STATES
------

Default (collapsed)
  Row shows normally. Content panel is hidden (max-height: 0).
  Toggle is checked (ON). Count reflects active total.

Expanded
  .is-expanded on .acc-wrap.
  Row border-radius flattens at bottom. Border color becomes #b1d1f6.
  Content panel gains .is-open (max-height: 500px, padding: 16px).
  Chevron rotates 180°.

Disabled (toggle off)
  .is-disabled on .acc-wrap.
  .acc__body, .acc__count, .acc__drag: opacity 0.42.
  .acc__drag and .acc__btn: pointer-events none.
  Row: cursor default, no hover background change.
  Toggle itself remains fully interactive (not dimmed) so the user can re-enable.

Hover
  Row background transitions to #eef3f8.
  Border color transitions to #c0cfe0.
  Inner expand button circle transitions to #cdd8eb.


BEHAVIOR
--------

Expand / Collapse
  Clicking the row OR the expand button toggles the open state.
  Use event.stopPropagation() on child elements (drag handle, toggle label,
  expand button) to prevent double-firing.

  JavaScript helper:
    function accToggle(id) {
      var wrap    = document.getElementById(id);
      var content = document.getElementById(id + '-c');
      var open    = wrap.classList.toggle('is-expanded');
      if (content) content.classList.toggle('is-open', open);
    }

Enable / Disable
  Toggle's onchange handler calls accSetDisabled.
  When disabled: expand and drag interactions are blocked; toggle still works.

  JavaScript helper:
    function accSetDisabled(id, disabled) {
      document.getElementById(id).classList.toggle('is-disabled', disabled);
    }

Drag to Reorder
  The drag handle icon signals reorderability. Wire up a drag-and-drop library
  (e.g. SortableJS) or the native HTML5 draggable API on .acc-wrap elements.
  Use stopPropagation() on .acc__drag mousedown/touchstart to avoid triggering
  the row expand click.


HTML STRUCTURE
--------------

<div class="acc-wrap" id="acc-0">           <!-- state container -->

  <div class="acc" onclick="accToggle('acc-0')">

    <!-- ① Drag handle -->
    <div class="acc__drag" onclick="event.stopPropagation()">
      <!-- 6-dot SVG -->
    </div>

    <!-- ② Toggle switch -->
    <label class="acc__toggle" onclick="event.stopPropagation()">
      <input type="checkbox" checked
             onchange="accSetDisabled('acc-0', !this.checked)">
      <span class="acc__track"></span>
      <span class="acc__thumb"></span>
    </label>

    <!-- ③ Title / Description -->
    <div class="acc__body">
      <div class="acc__title">Google Ads</div>
      <div class="acc__desc">Search, Shopping &amp; Display campaigns</div>
    </div>

    <!-- ④ Count (optional) -->
    <span class="acc__count">4/4</span>

    <!-- ⑤ Expand button -->
    <button class="acc__btn"
            onclick="event.stopPropagation(); accToggle('acc-0')">
      <div class="acc__btn-inner">
        <!-- chevron SVG (rotates when expanded) -->
      </div>
    </button>

  </div>

  <!-- ⑥ Content panel -->
  <div class="acc__content" id="acc-0-c">
    <!-- any content (rows, forms, lists…) -->
  </div>

</div>

Notes:
  • The ID suffix used for the content panel must be the row ID + "-c"
    so the toggle helpers can locate it automatically.
  • Wrap multiple accordion rows in a flex column with gap: 8px.
  • Do NOT give .acc-wrap.is-disabled .acc an onclick — remove it when disabled
    or skip adding it for rows that start in the disabled state.

ACCESSIBILITY
-------------
• The toggle input is a native <input type="checkbox"> — screen readers announce
  it correctly without extra ARIA.
• The expand <button> should have aria-expanded="true/false" wired in JS for full
  screen reader support.
• The .acc__drag element should have role="button" tabindex="0" and keyboard
  drag-and-drop support if reordering must be keyboard-accessible.
• Minimum touch target size: 32 × 32 px (satisfied by .acc__btn and .acc__drag).
